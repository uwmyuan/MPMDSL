Here is the dependency of basic algorithms for MILP: Branch&Price&Cut(MILP)<-Branch&Bround(MILP)<-Simplex(LP)
Simplex for LP may be a good start.
From optimization theory, we know the linear constraints form a polyhedron in the solution space and the optimal solution is on one of the vertex or edge or face. Simplex by Dantzig (1964) is basically transversing the vertices of the polyhedron to find the optimal solution or unbounded. It includes mostly matrix operations and not strongly polynomial.
Please refer to Page 220, the attached chapter for a formal algorithm.
Please refer to the hyperlink for a solid example: http://www.trentu.ca/academic/math/bz/341-l06.pdf .

For the CPU implementation, please refer to GNU GLPK for open source implementation (our first ast can be solved with this library) and the repo https://github.com/ehsanmok/spark-lp for the cluster based implementation code of interior point method (not simplex) with spark.
