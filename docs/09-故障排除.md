# ORMDSL 故障排除指南

本文档帮助解决使用ORMDSL时遇到的常见问题。

## 1. 安装问题

### 1.1 Gurobi 许可证问题

#### 问题: `LicenseError` 或 `GRBException`

```
gurobipy.gurobipy.GRBException: License error
```

#### 解决方案

```bash
# 1. 检查许可证文件
cat ~/.gurobi/gurobi.lic

# 2. 重新获取许可证
grbgetkey <your-license-key>

# 3. 设置许可证路径
export GRB_LICENSE_FILE=/path/to/gurobi.lic  # Linux/macOS
set GRB_LICENSE_FILE=C:\path\to\gurobi.lic  # Windows

# 4. 学术许可证验证
python -c "import gurobipy; print(gp.gurobi.version())"
```

### 1.2 Python 版本不兼容

#### 问题: `SyntaxError` 或模块导入失败

```
ModuleNotFoundError: No module named 'gurobipy'
```

#### 解决方案

```bash
# 检查Python版本
python --version

# 确保使用正确的Python
which python
which pip

# 重新安装
pip uninstall gurobipy
pip install gurobipy

# 使用虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/macOS
# 或
venv\Scripts\activate  # Windows
pip install gurobipy
```

## 2. 模型构建问题

### 2.1 模型不可行 (Infeasible)

#### 问题: `GRB.OPTIMAL` 状态为 `GRB.INFEASIBLE`

```
Optimization ended with status: 3
```

#### 诊断步骤

```python
import gurobipy as gp

model.optimize()

if model.status == GRB.INFEASIBLE:
    # 计算不可行约束
    model.computeIIS()
    
    print("不可行约束:")
    for c in model.getConstrs():
        if c.iisconstr:
            print(f"  - {c.constrName}")
    
    print("\n不可行变量:")
    for v in model.getVars():
        if v.iisub:
            print(f"  - {v.varName}")
```

#### 解决方案

1. **检查约束正确性**
```python
# 错误的约束
model.addConstr(x >= 10)  # 应该是 x <= 10
model.addConstr(x + y >= 100)  # 应该是 <=

# 检查数据
print(f"x范围: [{x.lb}, {x.ub}]")
print(f"y范围: [{y.lb}, {y.ub}]")
```

2. **放松约束找出冲突**
```python
# 添加人工变量 (惩罚项)
model.setParam("IntFeasTol", 1e-6)
model.setParam("FeasibilityTol", 1e-6)

# 或者使用约束松弛
def add_relaxed_constraints(model):
    for c in model.getConstrs():
        c.slack  # 检查松弛量
```

3. **常见不可行原因**
- 矛盾约束: `x >= 10` 且 `x <= 5`
- 整数约束过严
- 数据输入错误

### 2.2 模型无界 (Unbounded)

#### 问题: `GRB.UNBOUNDED`

```
Optimization ended with status: 5
```

#### 诊断步骤

```python
if model.status == GRB.UNBOUNDED:
    # 检查无界变量
    model.computeIIS()
    
    print("无界变量:")
    for v in model.getVars():
        if v.iisub:
            print(f"  - {v.varName}")
```

#### 解决方案

1. **设置变量边界**
```python
# 添加合理的上下界
x = model.addVar(lb=0, ub=10000, vtype=GRB.CONTINUOUS)
```

2. **检查目标函数**
```python
# 确保目标函数方向正确
model.setObjective(cost, GRB.MINIMIZE)  # 不能最小化无界方向
```

### 2.3 数值问题

#### 问题: `GRB.NUMERIC` 或解不稳定

```
Warning: numeric difficulties
```

#### 解决方案

```python
# 调整数值参数
model.Params.NumericFocus = 3  # 提高数值稳定性
model.Params.ScaleFlag = 2     # 更积极的缩放

# 避免大系数差异
# 约束: 1000000*x <= 1 改为: x <= 0.000001

# 归一化数据
for i in range(n):
    normalized_data[i] = original_data[i] / max_value
```

## 3. 求解性能问题

### 3.1 求解时间过长

#### 问题: 模型运行超过预期时间

#### 诊断步骤

```python
# 检查模型规模
print(f"变量数: {len(model.getVars())}")
print(f"约束数: {len(model.getConstrs())}")
print(f"非零系数: {model.NumNZs}")

# 检查LP松弛
model.optimize()
if model.status == GRB.TIME_LIMIT:
    print(f"当前目标值: {model.objVal}")
    print(f"最优下界: {model.objBound}")
    print(f"间隙: {100 * model.MIPGap}%")
```

#### 解决方案

1. **设置时间限制**
```python
model.Params.TimeLimit = 300  # 5分钟
```

2. **调整求解策略**
```python
# 使用对偶单纯形
model.Params.Method = 1

# 调整割平面
model.Params.Cuts = 1

# 启用启发式
model.Params.Heuristics = 0.8
```

3. **模型简化**
```python
# 预求解
model.Params.Presolve = 2

# 聚合相似约束
model.feasRelax(1, True, False, True)
```

### 3.2 最优性间隙过大

#### 问题: 求解器提前停止，间隙过大

```
MIPGap: 10%
```

#### 解决方案

```python
# 收紧间隙容忍度
model.Params.MIPGap = 0.001  # 0.1%

# 增加求解时间
model.Params.TimeLimit = 7200

# 调整节点选择策略
model.Params.NodeFileStart = 3  # 早点写入磁盘
model.Params.NodeMethod = 2     # 使用对偶单纯形
```

## 4. 代码错误

### 4.1 常见Python错误

#### 索引错误

```python
# 错误
for i in range(n):
    for j in range(m):
        cost[i, j] = data[i][j]  # 假设data是列表

# 正确
for i in range(n):
    for j in range(m):
        cost[i, j] = data.iloc[i, j]  # 使用pandas DataFrame
```

#### 变量作用域错误

```python
# 错误
def create_model():
    x = model.addVar()  # x是局部变量
    return model

x_solution = model.getVarByName("x")  # 在函数外无法访问

# 正确
def create_model():
    model = gp.Model()
    model._x = model.addVar()  # 保存引用
    return model

model = create_model()
x = model.getVarByName("x")
```

### 4.2 约束定义错误

#### 约束两边都是变量

```python
# 错误
model.addConstr(x, GRB.EQUAL, y)  # x == y 语法错误

# 正确
model.addConstr(x == y)
model.addConstr(x - y == 0)
model.addConstr(x >= y)
```

#### 错误的集合推导

```python
# 错误
model.addConstrs(x[i] for i in items)  # 需要索引

# 正确
for i in items:
    model.addConstr(x[i] <= capacity)
    
# 或者
model.addConstrs(x[i] <= capacity[i] for i in items)
```

## 5. 求解结果问题

### 5.1 解不符合预期

#### 诊断步骤

```python
model.optimize()

if model.status == GRB.OPTIMAL:
    print("变量值:")
    for v in model.getVars():
        if v.x > 0:
            print(f"  {v.varName} = {v.x}")
    
    print("\n约束检查:")
    for c in model.getConstrs():
        lhs = model.getRow(c)
        slack = c.slack
        if slack < -1e-6:  # 约束被违反
            print(f"  约束违反: {c.constrName} (slack={slack})")
```

#### 常见原因

1. **约束定义错误**
```python
# 应该 <= 但写成了 >=
model.addConstr(x + y <= 10)  # 检查这是否是本意
```

2. **数据错误**
```python
# 检查输入数据
print(cost_matrix)
print(capacity_values)
```

3. **变量类型错误**
```python
# 应该是整数但设为连续
x = model.addVar(vtype=GRB.CONTINUOUS)  # 应该是 INTEGER
```

### 5.2 目标函数值为负

#### 诊断

```python
if model.objVal < 0:
    print("目标函数值为负!")
    print("检查目标函数定义")
    
    # 打印目标表达式
    print(model.getObjective())
```

#### 解决方案

```python
# 如果本意是最小化成本，且成本为正
# 确保成本系数正确
model.setObjective(
    gp.quicksum(cost[i] * x[i] for i in items),
    GRB.MINIMIZE
)

# 如果使用惩罚项，确保符号正确
# 惩罚项: + M * violation
```

## 7. 获取帮助

### 7.1 调试清单

```python
# 完整调试清单
def debug_model(model):
    """模型调试函数"""
    results = {
        "status": model.status,
        "vars": len(model.getVars()),
        "constrs": len(model.getConstrs()),
        "nz": model.NumNZs,
    }
    
    if model.status == GRB.OPTIMAL:
        results["objval"] = model.objVal
        results["runtime"] = model.runtime
        results["gap"] = model.MIPGap
    elif model.status == GRB.INFEASIBLE:
        model.computeIIS()
        results["iis_constraints"] = [c.constrName for c in model.getConstrs() if c.iisconstr]
    elif model.status == GRB.UNBOUNDED:
        model.computeIIS()
        results["iis_vars"] = [v.varName for v in model.getVars() if v.iisub]
    
    return results
```

## 8. 错误代码参考

| 状态码 | 名称 | 说明 |
|--------|------|------|
| 1 | LOADED | 模型已加载 |
| 2 | OPTIMAL | 最优解 |
| 3 | INFEASIBLE | 不可行 |
| 4 | INF_OR_UNBD | 不可行或无界 |
| 5 | UNBOUNDED | 无界 |
| 6 | CUTOFF | 被截断 |
| 7 | ITERATION_LIMIT | 达到迭代限制 |
| 8 | NODE_LIMIT | 达到节点限制 |
| 9 | TIME_LIMIT | 达到时间限制 |
| 10 | SOLUTION_LIMIT | 达到解数量限制 |
| 11 | INTERRUPTED | 被中断 |
| 12 | NUMERIC | 数值问题 |
| 13 | SUBOPTIMAL | 次优解 |
