# ORMDSL 项目概述

## 1. 项目简介

**ORMDSL (Object-Relational Modeling for Discrete Optimization Domain Specific Language)** 是一个用于数学规划建模的领域特定语言（Domain Specific Language, DSL）。

### 1.1 设计目标

ORMDSL 旨在解决数学优化建模中的三个核心问题：

| 问题 | 描述 | ORMDSL解决方案 |
|------|------|---------------|
| **阻抗不匹配** | 问题域概念与求解器API之间存在鸿沟 | 约束集类直接映射领域概念 |
| **公式鸿沟** | 学术论文公式与代码实现难以对应 | 声明式语法接近数学符号 |
| **代码复用** | 相似问题需要重复编写约束 | 模块化约束集可复用 |

### 1.2 适用场景

✅ **推荐使用**
- 学术研究中的优化模型快速验证
- 需要生成多格式输出（AMPL/Python/LaTeX）的场景
- 追求高度模块化和代码复用的项目
- 问题域概念明确（如物流、调度、网络流）

⚠️ **谨慎使用**
- 大规模工业级优化问题（推荐使用Pyomo/GAMS）
- 需要复杂约束编程/启发式算法的混合问题
- 实时性能要求极高的场景

## 2. 核心特性

### 2.1 约束集类 (Constrained Set Class)

约束集类（CS Class）是ORMDSL的核心创新。它将问题域的基本概念直接建模为类：

```python
# 传统方式 (Pyomo风格)
model.x = Var(I, J)
model.c = Constraint(expr=sum(model.x[i,j] for i in I for j in J) <= W)

# ORMDSL方式 (领域驱动)
class Vessel extends CS:
    arrival_time: DecisionVariable
    departure_time: DecisionVariable
    berth: DecisionVariable

class Berth extends CS:
    capacity: Parameter
    available_time: Parameter
```

### 2.2 符号自动重写

ORMDSL 支持自动进行符号级别的优化：

```python
# 蕴含约束的自动Big-M重写
# DSL层面：if (x == 1) then (y >= a)
# 自动重写为：y >= a * x
```

### 2.3 多语言代码生成

一个模型，多种输出：

| 目标 | 用途 | 生成器 |
|------|------|--------|
| AMPL | 学术研究，求解器接口 | `AST2AMPL.scala` |
| Gurobi | 工业级求解 | `IR2Gurobi.scala` |
| Python | 生产环境 | `ormdsl-py/` |
| LaTeX | 论文公式展示 | `AST2Tex.scala` |

### 2.4 声明式语法

```python
# 接近数学符号的表达
MIN customers.FOLD(+, c → c.demand * travel[c, allocation(c)])

# 约束声明
CONSTRAINT size(established_warehouse) == P
```

## 4. 与现有工具对比

### 4.1 功能对比

| 特性 | ORMDSL | Pyomo | GAMS | AMPL |
|------|--------|-------|------|------|
| **抽象层次** | 领域对象 | 变量/约束 | 代数表达式 | 代数表达式 |
| **约束集类** | ✅ 原生支持 | ⚠️ 需插件 | ❌ | ❌ |
| **符号重写** | ⚠️ 部分完成 | ❌ | ⚠️ 有限 | ❌ |
| **多语言生成** | ✅ 4种 | ❌ | ❌ | ❌ |
| **LaTeX输出** | ✅ | ❌ | ❌ | ⚠️ |
| **学习曲线** | 中等 | 中等 | 陡峭 | 陡峭 |
| **社区生态** | 小 | 活跃 | 成熟 | 成熟 |
| **文档完善度** | ⚠️ 发展中 | ✅ 完善 | ✅ 完善 | ✅ 完善 |

### 4.2 ORMDSL 的独特优势

1. **领域驱动设计** - 用问题域术语而非求解器概念建模
2. **学术友好** - 降低论文公式到代码的转换成本
3. **多格式输出** - 一套模型，学术/生产多端可用
4. **模块化约束** - 约束可复用、可组合
